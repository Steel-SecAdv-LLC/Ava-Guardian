### ML-DSA-65 (Module-Lattice Digital Signature Algorithm)

**Standard**: NIST FIPS 204 (formerly Dilithium)
**Type**: Post-Quantum Lattice-based
**Security**: NIST Level 3 (192-bit quantum security)

**Advantages**:
- âœ… Quantum-resistant (safe against Shor's algorithm)
- âœ… NIST-approved standard
- âœ… Relatively small signatures compared to hash-based schemes
- âœ… Fast verification

**Trade-offs**:
- âš ï¸ Larger keys than classical schemes
- âš ï¸ Slower signing than Ed25519

### SPHINCS+-256f (Stateless Hash-Based Signatures)

**Standard**: NIST PQC Round 3 Alternate Candidate
**Type**: Post-Quantum Hash-based
**Security**: 256-bit classical and quantum

**Advantages**:
- âœ… Quantum-resistant
- âœ… Stateless (no state management required)
- âœ… Conservative security assumptions (hash functions only)
- âœ… Tiny public keys (64 bytes)

**Trade-offs**:
- âš ï¸ Very large signatures (49856 bytes)
- âš ï¸ Slower signing than lattice-based schemes

### Ed25519 (Edwards-curve Digital Signature Algorithm)

**Standard**: RFC 8032
**Type**: Classical Elliptic Curve
**Security**: 128-bit classical (broken by quantum)

**Advantages**:
- âœ… Very fast signing and verification
- âœ… Tiny signatures (64 bytes)
- âœ… Deterministic (no nonce reuse vulnerability)
- âœ… Wide deployment and testing

**Trade-offs**:
- âš ï¸ Vulnerable to quantum computers (Shor's algorithm)
- âš ï¸ Will need migration in 5-15 years

### Hybrid Mode (Ed25519 + ML-DSA-65)

**Best of both worlds**: Combines classical and post-quantum

**Advantages**:
- âœ… Secure if either algorithm is unbroken
- âœ… Smooth migration path
- âœ… Backward compatibility

**Use When**:
- Transitioning to PQC
- Maximum security is required
- Backward compatibility needed
